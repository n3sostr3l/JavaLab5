## 3. Архитектура и сетевое взаимодействие (NIO)
- **Независимость процессов:** Выход клиентского приложения не должен влиять на работу сервера.
- **Рефакторинг команд:** Перенести реализацию всех команд из общей папки (`general`) в папку сервера (`server`), так как клиенту не требуется знать о внутренней логике выполнения команд.
- **Использование селекторов:** Перевести сетевое взаимодействие на неблокируемый ввод-вывод (`java.nio.channels.Selector`).
- **Модель обмена данными:** Коллекция хранится в памяти сервера. Клиенты инициируют изменения. При аварийном завершении сервера данные записываются во временный файл.
- **Управление командами:** Обеспечить корректную передачу и обработку команд управления (`exit`, `save`) от клиента к серверу.

## 4. Специальные возможности (Админ-клиент)
- **Создание админ-клиента:** Реализовать отдельный вид клиента ("Админ") с расширенными правами доступа.
- **Эксклюзивные команды:**
    - `exit_server`: Команда, полностью завершающая процесс сервера.
    - `save`: Ручное сохранение текущего состояния коллекции в файл для обеспечения персистентности между перезапусками сервера.
