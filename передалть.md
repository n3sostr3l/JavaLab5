# Лабораторная работа №6: Список задач на доработку

- **Восстановление сессии:** При старте программы предлагать пользователю восстановить коллекцию, если предыдущая сессия не была сохранена.

Это должно быть реализовано так, для клиента хранится в одном файле последняя не восстановленная сессия и в другом файле последняя восстановленная сессия (если еще не было - то пустой файл), Если пользователь выбрал восстановить, то подается коллекция из файла с последней не восстановленной и из другого файла в противном случае.

## 3. Архитектура и сетевое взаимодействие (NIO)
- **Независимость процессов:** Выход клиентского приложения не должен влиять на работу сервера.

- **Использование селекторов:** Перевести сетевое взаимодействие на неблокируемый ввод-вывод (`java.nio.channels.Selector`).

## 4. Специальные возможности (Админ-клиент)
- **Создание админ-клиента:** Реализовать отдельный вид клиента ("Админ") с расширенными правами доступа. У него должны быть только эти команды:
    - `exit_server`: Команда, полностью завершающая процесс сервера. Перед запуском спрашивается подтверждение: "Вы уверены, что хотите завершить работу сервера? (y/n). Все клиенты не смогут восстановить текущую сессию. Введите save для сохранения коллекции"
    - `save`: Сохранение всей текущей коллекции, несохраненную сессию в файл с не восстановленной сессией для каждого клиента и все.

    Пока что отталкиваемся от того, что клиент всего один, то есть одна коллекция на всех.
